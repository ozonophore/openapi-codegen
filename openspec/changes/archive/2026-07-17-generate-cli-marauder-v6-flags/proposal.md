## Почему

Marauder Platform V6 вводит пять новых опций на уровне root-конфига (`workspaceReport`, `trafficSplitter`, `swarm`, `preAnalyze`, `reuseMode`) в коммитах `9adadcd`, `7727a79`, `8071efe` и `efba83f`. Сейчас эти опции доступны только через `openapi.config.json`; в CLI-команде `generate` нет соответствующих флагов, нет валидации новых полей и нет примера конфига с их использованием. Разработчики, предпочитающие CLI-подход или CI-пайплайны, не могут включить эти функции без редактирования конфигурационного файла.

## Что меняется

- **Новые CLI-флаги** для `openapi-codegen generate`:
  - `--workspace-report` — включает `workspaceReport` со значениями по умолчанию
  - `--traffic-splitter` — включает `trafficSplitter` со значениями по умолчанию
  - `--swarm` — включает `swarm` (AvatarSwarm-манифест) со значениями по умолчанию
  - `--pre-analyze` — включает `preAnalyze` (предгенерационный cross-spec анализ)
  - `--reuse-mode <copy|auto-group>` — устанавливает `reuseMode`
- **Схема валидации CLI** обновляется для принятия и проверки всех пяти новых флагов (типы, допустимые значения, правила взаимного исключения, ограничения root-only)
- **Новый пример** `openapi.config.json` в директории `example/`, демонстрирующий все новые опции Marauder V6 вместе с пояснениями

## Возможности

### Новые возможности

- `generate-cli-marauder-flags`: CLI-флаги для `generate`, соответствующие пяти новым root-уровневым опциям конфига Marauder V6
- `generate-cli-validation`: правила Zod/yargs-валидации для новых CLI-флагов (проверка типов, допустимые enum-значения, приведение из булевого сокращения)
- `example-marauder-config`: аннотированный пример `openapi.config.json`, показывающий `workspaceReport`, `trafficSplitter`, `swarm`, `preAnalyze` и `reuseMode` в реалистичном сценарии multi-spec монорепозитория

## Влияние

- `src/cli/generate.ts` (или аналогичная точка входа CLI) — определения флагов
- Модуль схемы валидации CLI — новые правила для полей
- Директория `example/` — новый файл `openapi.config.json`
- Ссылки на опции команды `generate` в README/документации (обновление документации, без breaking changes)
- Никаких изменений в runtime-поведении за пределами того, что уже описано в `openspec/research/PDD.md` для каждой функции Marauder
